# Superstore Sales Dashboard 📦📈

An interactive Power BI dashboard for visualizing and analyzing retail sales data from a Superstore. This project enables stakeholders to understand key performance metrics like sales, profit, customer segments, and regional performance through compelling data visuals.

---

## 📂 Project Overview

The **Superstore Sales Dashboard** provides a comprehensive view of a fictional retail company's performance across multiple dimensions such as Region, Category, Customer Segment, and Order Date. It is aimed at helping decision-makers optimize operations and boost profitability by identifying trends, patterns, and outliers.

---

## 📊 Key Features

- **Sales & Profit Trends**: Analyze how sales and profits perform over time.
- **Category & Sub-Category Insights**: Deep dive into product performance.
- **Geographic Analysis**: Visualize region-wise and state-wise sales/profit on maps.
- **Customer Segmentation**: Breakdown of sales by Customer Segment.
- **Top Performers**: Identify top-selling products and high-revenue customers.
- **Interactive Slicers**: Filter data by Region, Category, Year, Segment, etc.

---

## 📁 Dataset

- **File**: `superstore_sales.csv` (included inside the `.zip`)
- **Source**: Based on a classic sample dataset frequently used for BI and Tableau demos
- **Size**: ~10,000+ records
- **Columns Include**:
  - Order ID, Order Date, Ship Date
  - Customer ID, Segment, Region, State, City
  - Product Category/Sub-Category
  - Sales, Quantity, Discount, Profit

## 🧠 Key Insights

- The **West** region generates the highest profit despite lower sales compared to **East**.
- **Technology** products have the highest average profit margin.
- Significant losses occur with high discount levels in **Office Supplies**.
- **California** and **New York** are top states by revenue.

## 🚀 How to Use

1. Clone this repository.
2. Unzip the project files.
3. Open the `.pbix` file using Power BI Desktop.
4. Ensure the dataset path is correctly mapped. Use **Transform Data > Edit Queries** if needed.
5. Interact with the dashboard using slicers and visuals.
